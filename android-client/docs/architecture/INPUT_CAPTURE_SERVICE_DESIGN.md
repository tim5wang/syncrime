# 输入采集服务设计文档

## 📋 概述

实现 Android AccessibilityService 来采集用户输入内容，用于智能推荐和同步。

## ⚠️ 隐私与安全原则

1. **用户知情** - 明确告知用户采集内容和用途
2. **最小化采集** - 只采集必要信息
3. **本地加密** - 敏感数据加密存储
4. **用户控制** - 随时可以开启/关闭采集
5. **敏感过滤** - 自动过滤密码、银行卡等敏感信息

## 🔐 敏感信息过滤规则

### 自动过滤的字段类型
1. **密码字段** - `inputType == TYPE_TEXT_VARIATION_PASSWORD`
2. **可见密码** - `inputType == TYPE_TEXT_VARIATION_VISIBLE_PASSWORD`
3. **Web 密码** - `inputType == TYPE_TEXT_VARIATION_WEB_PASSWORD`
4. **邮箱地址** - 正则匹配
5. **电话号码** - 正则匹配
6. **银行卡号** - 正则匹配 (16-19 位数字)
7. **身份证号** - 正则匹配

### 敏感词过滤
- 自定义敏感词库
- 支持正则表达式
- 支持分类标记

## 📱 用户体验

### 权限请求流程
1. 首次启动 → 显示说明对话框
2. 用户同意 → 跳转到无障碍设置
3. 开启服务 → 返回应用
4. 状态监控 → 服务异常时提醒

### 状态指示
- 通知栏常驻状态
- 采集状态图标
- 一键暂停/恢复

## 📊 性能优化

1. **事件去重** - 避免重复处理相同事件
2. **异步处理** - 不阻塞 UI 线程
3. **批量处理** - 合并短时间内的多次输入
4. **内存管理** - 及时释放不用的资源

## 🔧 技术实现

### 核心组件
- `InputCaptureService` - AccessibilityService 实现
- `InputFilterManager` - 敏感信息过滤
- `InputProcessor` - 输入内容处理
- `CaptureNotificationManager` - 通知管理

### 关键配置
- `accessibility_service_config.xml` - 服务配置
- 监听事件类型：`TYPE_VIEW_TEXT_CHANGED`
- 监听反馈类型：`FEEDBACK_GENERIC`
